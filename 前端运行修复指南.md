# 前端项目运行问题修复

## 问题原因
前端无法运行是因为 `node_modules` 中的依赖包不完整,缺少 `which` 模块。

## 已执行的修复步骤

### 1. 清理旧依赖
```bash
cd D:\小程序开发练习\admin-shop-frontend
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
```

### 2. 重新安装依赖
```bash
npm install
```

**当前状态**: ✅ 正在安装中...

## 常见问题

### 1. npm install 很慢或卡住
**解决方案**: 使用淘宝镜像
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 2. 端口被占用 (EADDRINUSE)
**解决方案**: 
```bash
# Windows下查找占用8081端口的进程
netstat -ano | findstr :8081
# 杀死进程 (替换PID为实际进程ID)
taskkill /F /PID <PID>
```

或者修改 `manifest.json` 中的端口号

### 3. 仍然提示缺少模块
**解决方案**: 删除后重新安装
```bash
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install --force
```

### 4. Node.js版本不兼容
当前项目需要 Node.js 14-18 版本
检查版本:
```bash
node -v
```

如果版本过高(如v24),建议使用nvm切换到低版本:
```bash
# 安装nvm-windows后
nvm install 18
nvm use 18
```

## 验证是否成功
