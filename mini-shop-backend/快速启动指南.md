已从 `C:\Users\yao\Desktop\新cookie.txt` 文件中提取并配置以下信息：

- ✅ Cookie 信息

## 🚀 启动步骤

### 1. 启动后端服务

```bash
cd D:\Java项目\mini-shop\mini-shop-backend
mvn spring-boot:run
```

或者在IDE中运行 `MiniShopApplication.java`

### 2. 验证服务启动

服务将在 http://localhost:8090 启动

### 3. 测试接口

#### 方式一：使用curl命令
```bash
curl -X GET "http://localhost:8090/api/douyin/order/detail?orderToken=YOUR_ORDER_TOKEN&orderId=YOUR_ORDER_ID"
```

#### 方式二：浏览器访问
```
http://localhost:8090/api/douyin/order/detail?orderToken=YOUR_ORDER_TOKEN&orderId=YOUR_ORDER_ID
```

#### 方式三：使用Postman
- Method: GET
- URL: http://localhost:8090/api/douyin/order/detail
- Params:
  - orderToken: YOUR_ORDER_TOKEN
  - orderId: YOUR_ORDER_ID

## 📋 接口说明

**接口地址**: `GET /api/douyin/order/detail`

**请求参数**:
- `orderToken` (必填): 下单token
- `orderId` (必填): 订单号

**响应格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 抖音订单详情数据
  }
}
```

## ⚠️ 注意事项

1. **替换实际参数**: 测试时需要将 `YOUR_ORDER_TOKEN` 和 `YOUR_ORDER_ID` 替换为实际的订单token和订单号

2. **Cookie时效性**: Cookie有时效性，如果查询失败，需要从新的cookie.txt文件更新配置

3. **网络连接**: 确保服务器可以访问抖音API

4. **频率限制**: 注意调用频率，避免被限制

## 🔧 配置文件位置

所有配置都在: `mini-shop-backend/src/main/resources/application.yml`

抖音API配置段:
```yaml
douyin:
  api:
    order-detail-url: https://api.douyin.com/api/order/detail
    cookie: "..."
    device-token: "..."
    xtt-token: "..."
    user-agent: "..."
```

## ✨ 技术实现

- 使用 Spring Boot RestTemplate 发送HTTP请求
- 配置信息通过 @ConfigurationProperties 自动注入
- 支持动态配置更新
- 完善的错误处理机制
