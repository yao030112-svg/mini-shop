# 管理员账号设置指南

## 问题说明
登录时提示"管理员账号不存在"，是因为数据库中没有正确的管理员账号。

## 解决方案

### 方案1：执行SQL脚本（推荐）

1. 打开MySQL客户端或Navicat等工具
2. 连接到你的数据库
3. 执行以下SQL：

```sql
-- 方法A: 如果已存在nickname为'admin'的记录，更新它
UPDATE user 
SET password = '$2a$10$lBdmQ8hoq3hPnghczqW.4.kqPYm7UH4sVohYGTxXdBFi8SOnKq/66',
    role = 1,
    status = 1,
    deleted = 0
WHERE nickname = 'admin';

-- 方法B: 如果不存在，插入新记录
INSERT INTO user (openid, nickname, phone, password, role, status, deleted, create_time, update_time)
VALUES (
    'admin_openid_001',
    'admin',
    '13800000000',
    '$2a$10$lBdmQ8hoq3hPnghczqW.4.kqPYm7UH4sVohYGTxXdBFi8SOnKq/66',
    1,
    1,
    0,
    NOW(),
    NOW()
);
```

### 方案2：使用项目提供的SQL文件

执行项目根目录下的SQL文件：
- `create_admin.sql` - 创建/更新管理员账号
- `mini-shop-backend/src/main/resources/sql/data.sql` - 包含初始数据（已有管理员账号）

## 登录信息

创建成功后，使用以下信息登录：

- **用户名**: `admin`
- **密码**: `admin123`

## 验证账号是否创建成功

执行以下SQL查询：

```sql
SELECT id, nickname, phone, role, status, deleted 
FROM user 
WHERE nickname = 'admin' AND role = 1;
```

应该返回一条记录，其中：
- `role = 1` (表示管理员)
- `status = 1` (表示正常状态)
- `deleted = 0` (表示未删除)

## 常见问题

### 1. 仍然提示"管理员账号不存在"
- 检查数据库中是否有 `nickname='admin'` 且 `role=1` 的记录
- 确认没有拼写错误（注意大小写）

### 2. 提示"密码错误"
- 确认密码是 `admin123`
- 确认password字段存储的是BCrypt加密后的值

### 3. 提示"未认证/Token过期"
- 先重新登录获取新的token
- 检查前端是否正确保存了token到localStorage

如果需要修改密码，可以使用在线BCrypt生成器生成新的加密值。
