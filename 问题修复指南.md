### 2. ❌ API接口404 - "/api/admin/stats"
**原因**: AdminController可能没有被正确扫描或路径配置问题

**检查步骤**:
1. 确认后端服务已启动
2. 检查AdminController是否存在且路径正确
3. 查看后端日志是否有错误

**相关文件**:
- Controller: `mini-shop-backend/src/main/java/com/minishop/controller/AdminController.java`
- Service: `mini-shop-backend/src/main/java/com/minishop/service/AdminService.java`

---

### 3. ⚠️ showLoading/hideLoading未配对使用
**原因**: 在某些页面中调用了showLoading但没有在finally或catch中调用hideLoading

**已修复**:
- ✅ `pages/admin/login.vue` - 已添加正确的showLoading/hideLogic配对

**需要检查的其他页面**:
- admin-shop-frontend/pages/admin/index.vue (第166、176行)
- 其他管理页面

**修复示例**:
```javascript
async loadData() {
  uni.showLoading({ title: '加载中...' })
  try {
    const res = await someApi()
    // 处理数据
    uni.hideLoading()
  } catch (e) {
    console.error('加载失败', e)
    uni.hideLoading()  // 确保在catch中也调用
  }
}
```

---

### 4. ❌ Token认证失败 - "未认证/Token过期"
**原因**: 
- Token已过期
- Token未正确保存
- 请求时未携带Token

**解决方案**:
1. 重新登录获取新Token
2. 检查Token是否正确保存到storage
3. 检查request.js中是否正确添加了Authorization header

**检查代码** (`api/request.js`):
```javascript
// 确保请求拦截器中添加了token
const token = uni.getStorageSync('token')
if (token) {
  header['Authorization'] = `Bearer ${token}`
}
```

## 快速修复步骤

### 第一步：创建管理员账号
1. 打开MySQL客户端
2. 连接到数据库
3. 执行 `create_admin.sql` 文件中的SQL语句
4. 验证账号是否创建成功

### 第二步：重启后端服务
```bash
cd mini-shop-backend
mvn clean spring-boot:run
```

### 第三步：清除前端缓存并重新登录
1. 清除浏览器localStorage
2. 刷新页面
3. 使用 `admin` / `admin123` 登录

### 第四步：检查API是否正常
打开浏览器开发者工具，查看Network标签：
- 确认请求URL正确
- 确认请求头中包含Authorization
- 确认响应状态码为200
