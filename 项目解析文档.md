# 迷你商城 - 项目详细解析

## 一、整体架构

```
用户操作手机/浏览器
       ↓
┌─────────────────────────────┐
│   前端 (uni-app + Vue 2)     │  ← 负责页面展示和用户交互
│   运行在: localhost:8081      │
└──────────────┬──────────────┘
               │ HTTP 请求 (JSON)
               ↓
┌─────────────────────────────┐
│   后端 (Spring Boot)         │  ← 负责业务逻辑和数据处理
│   运行在: localhost:8090      │
└──────────────┬──────────────┘
               │ SQL 查询
               ↓
┌─────────────────────────────┐
│   数据库 (MySQL)             │  ← 负责数据存储
│   数据库名: mini_shop         │
└─────────────────────────────┘
```

**前后端如何联络：**
1. 前端发起 HTTP 请求（如 GET http://localhost:8090/api/product/list）
2. 后端 Controller 接收请求，调用 Service 处理业务逻辑
3. Service 通过 Mapper 查询数据库
4. 数据库返回数据 → Service 处理 → Controller 封装为 JSON 返回
5. 前端收到 JSON 数据，渲染到页面上

### 2.2 分层架构（从上到下）

```
Controller（控制器）→ 接收前端请求，返回响应
     ↓
Service（服务层）→ 处理业务逻辑（如：下单时检查库存、扣减库存）
     ↓
Mapper（数据访问层）→ 执行 SQL 操作数据库
     ↓
数据库表
```

### 3.2 API 请求层 (`api/`)

**核心文件 `api/request.js`：**
```javascript
// 封装所有 HTTP 请求的基础方法
const BASE_URL = 'http://localhost:8090'  // 后端地址

function request(options) {
    const token = uni.getStorageSync('token')  // 从本地取 Token
    
    return uni.request({
        url: BASE_URL + options.url,
        method: options.method,
        data: options.data,
        header: {
            'Authorization': token ? `Bearer ${token}` : ''  // 带上 Token
        }
    }).then(res => {
        if (res.data.code === 401) {
            // Token 过期，跳转登录页
            uni.navigateTo({ url: '/pages/login/index' })
        }
        return res.data
    })
}
```

**各模块 API 文件：**

## 四、一个完整请求的流程（以"加入购物车"为例）

```
第1步：用户在商品详情页点击"加入购物车"按钮
       ↓
第2步：前端 pages/product/detail.vue 调用方法
       handleAddToCart() → addToCart(skuId)
       ↓
第3步：前端 api/cart.js 发送 HTTP 请求
       POST http://localhost:8090/api/cart/add
       请求体: { "skuId": 1 }
       请求头: Authorization: Bearer eyJhbGci...（Token）
       ↓
第4步：后端 JwtInterceptor 拦截请求
       → 验证 Token 有效
       → 解析出 userId=2
       → 存入 UserContext
       → 放行
       ↓
第5步：后端 CartController.add() 接收请求
       Long userId = UserContext.getCurrentUserId()  // 得到 userId=2
       cartService.addToCart(userId, skuId)
       ↓
第6步：后端 CartService.addToCart() 处理业务逻辑
       → skuMapper.findById(skuId)  // 查 SKU 是否存在
       → 检查库存是否 > 0
       → cartMapper.findByUserIdAndSkuId(userId, skuId)  // 查是否已在购物车
       → 如果已存在：cartMapper.updateQuantity(id, quantity+1)  // 数量+1
       → 如果不存在：cartMapper.insert(新条目)  // 新增一条
       ↓
第7步：后端 CartMapper 执行 SQL
       INSERT INTO cart (user_id, sku_id, quantity, selected) 
       VALUES (2, 1, 1, 1)
       ↓
第8步：数据库写入成功，逐层返回
       CartMapper → CartService → CartController
       → 返回 { code: 200, message: "success", data: null }
       ↓
第9步：前端收到响应
       api/cart.js 返回结果
       → detail.vue 显示 "已加入购物车" 提示
```

## 五、认证机制（Token 是怎么工作的）

```
1. 用户登录
   前端发送 code → 后端验证 → 生成 Token（包含 userId 和 role）→ 返回给前端

2. 前端存储 Token
   uni.setStorageSync('token', token)  // 存到手机本地

3. 后续每次请求都带上 Token
   请求头: Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

4. 后端验证 Token
   JwtInterceptor 解析 Token → 得到 userId → 知道是谁在操作

5. Token 过期（7天后）
   后端返回 401 → 前端跳转登录页 → 重新登录获取新 Token
```

## 六、数据库表之间的关系

```
user（用户）
  ├── cart（购物车）→ 用户有多个购物车条目
  ├── address（地址）→ 用户有多个收货地址
  └── orders（订单）→ 用户有多个订单
        └── order_item（订单项）→ 一个订单有多个商品

category（分类）
  └── product（商品）→ 一个分类下有多个商品
        ├── product_spec（规格维度）→ 如"颜色"、"尺码"
        │     └── spec_value（规格值）→ 如"红色"、"XL"
        └── sku（库存单位）→ 一个商品有多个 SKU
              └── sku_spec_value（SKU规格关联）→ 哪个 SKU 对应哪些规格值

banner（轮播图）→ 独立表，首页展示用
```

## 七、项目实现顺序

1. **先搭后端骨架** → 创建 Spring Boot 项目、配置数据库、建表
2. **实现认证模块** → JWT 生成/验证、拦截器、登录接口
3. **实现核心业务** → 商品 → 购物车 → 订单（按依赖顺序）
4. **实现管理功能** → 管理员的商品/订单/用户管理接口
5. **搭前端骨架** → 创建 uni-app 项目、配置路由、封装请求
6. **实现前端页面** → 首页 → 分类 → 详情 → 购物车 → 订单 → 个人中心
7. **实现管理后台** → 管理员登录和管理页面
8. **编写测试** → 属性测试验证业务逻辑正确性
